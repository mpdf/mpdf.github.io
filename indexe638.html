Using embedded font subsets

The standard usage of mPDF creates PDF files with embedded fonts. You can choose between embedding small font files with font information about characters from selected codepages e.g. win-1252. This makes small PDF files, but limits the characters that can be used in the document. Alternatively - in UTF-8 mode - large font files are embedded, resulting in large PDF files, but allowing a wide range of characters to display.

Introduced in version 4.x, mPDF will allow the option to embed a subset of a font (only those characters used in the document) - making all characters available, but with a small PDF file size. This is done by adding -s when instantiating the mpdf class e.g. $mpdf = new mPDF('utf-8-s');

mPDF requires extra files for each font in order to re-assemble a font subset. You will need to download the extra font file pack in order to use subsets. These files should be uploaded to the [path_to_mpdf_folder]/unifont/ folder on your server.

NB The additional .dat,  .dat.php,  and .uni2gn.php files for the basic font pack take up about 29MB file space when uncompressed.

Once these additional files have been uploaded, you should be able to create a PDF document with font subsets just by using the '-s'.

Using embedded font subsets will cause an increase in processing time to generate the file.

Installing additional font files

The additional font files required for embedding subsets are not included in the basic installation, and need to be downloaded separately. These files should be uploaded to your server in the [mpdf]/unifont/  directory.

  Basic files required Additional files required
for embedding subsets
mpdf/unifont/  directory
(UTF-8 font files)		 [fontname].php
[fontname].z
[fontname].ctg.z		 [fontname].dat
[fontname].dat.php
[fontname].uni2gn.php

NB Indic fonts contain an additional [fontname].volt.php file
