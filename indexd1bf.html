Upgrading 4.1.1 to 4.2 - Archived

If you have made changes to any of your configuration files etc. please keep a safe copy first. These may include:

  • config.php
  • config_cp.php
  • config_fonts.php
  • mpdf.css

Download the Upgrade .zip file and unzip it

  • Upload all of the included files to the server, maintaining the folders as they are
  • Set read & write permissions for the new /tmp/ folder
Changes in layout

Some of the changes in v4.2 may affect the layout of documents you have already developed. Some or all of the following will restore the previous layout to your documents.

Where CSS changes are suggested, you can either edit the $defaultCSS variable in the config.php file or you can edit the mpdf.css file for global changes; alternatively just make the changes for individual documents.

Where configuration variables changes are suggested, you can either edit the config.php file, or add to each script as required.

Line-height - space between lines has increased

Set $this->normalLineheight = 1.2; in config.php

or $this->defaultCSS['BODY']['LINE-HEIGHT'] => 1.2  in config.php

Vertical Space between block elements has reduced

Set $this->collapseBlockMargins = false; in config.php

Space before first block element at top of page has disappeared

Set $this->defaultCSS['BODY']['MARGIN-COLLAPSE'] => 'none'  in config.php

Images have no margin around them and/or are aligned differently to the text

Add to mpdf.css

img { margin: 0.83em 0; vertical-align: bottom; }

Tables have no/less space above or below

Add to mpdf.css

table { margin: 0.5em; }

Textarea is aligned different vertically

Add to mpdf.css

textarea { vertical-align: top; }
