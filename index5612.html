SetUserRights() - Archived

(mPDF >= 2.0, < 2.4)

SetUserRights â€“ Sets the PDF document permissions for Annotations or Forms

** SetUserRights was removed in mPDF 2.4 **

Description

void SetUserRights ( boolean $flag [, string $annots [, string $form [, string $signature ]]])

Sets the PDF document permissions for annotations, forms or signatures. Permissions for copying, printing and modifying the document are set with SetProtection().

Note: Annotations written into the PDF document will be seen by all users. They will only be able to move or delete them, and re-save the document if you SetUserRights() with the annots permissions set. If the file is encrpyted with SetProtection() ensure that the modify and annot-forms permissions are granted also.

Note: Active Forms are very experimental at present. If using, it is suggested that the full string of permissions are granted.

Note: This function/method was altered in mPDF 2.2 by capitalising the first letter of the name. As function/method names in PHP have hitherto been case-insensitive, this should not cause any problems, but it is recommended where possible to use the preferred spelling.

Parameters

flag

Set TRUE or FALSE to grant permissions to the user.
DEFAULT: TRUE

annots

This parameter sets the permission for Annotations.
A blank string should be passed to deny all permissions to the user.

Values (case-sensitive)
A string containing any or all of the following values concatenated into a string (with no spaces). The values included are those permissions allowed:
/Create
/Delete
/Modify
/Copy
/Import
/Export
DEFAULT: "/Create/Delete/Modify/Copy/Import/Export"

form

This parameter sets the permission for active Forms.
A blank string should be passed to deny all permissions to the user.

Values (case-sensitive)
A string containing any or all of the following values concatenated into a string (with no spaces). The values included are those permissions allowed:
/Add
/Delete
/FillIn
/Import
/Export
/SubmitStandalone
/SpawnTemplate
DEFAULT: "/Add/Delete/FillIn/Import/Export/SubmitStandalone/SpawnTemplate"

signature

This parameter sets the permission for Signatures.
A blank string should be passed to deny all permissions to the user.

Values (case-sensitive)
A string containing any or all of the following values concatenated into a string (with no spaces). The values included are those permissions allowed:
/Modify
DEFAULT: "/Modify"

Examples

Example #1

<?php

$mpdf=new mPDF();

// This sets sufficient rights for the user to modify your annotations
$mpdf->SetUserRights(true, '/Create/Delete/Modify/Copy/Import/Export');

// If you want to encrypt the file, include the necessary permissions
$mpdf->SetProtection(array('modify','annot-forms'), 'UserPassword', 'MyPassword');

$mpdf->WriteHTML('<p><annotation content="You can move this around or delete it" />Hallo World</p>');
$mpdf->Output('filename.pdf');

?>

Example #2


// To experiment with active Forms, use
$mpdf->SetUserRights();
