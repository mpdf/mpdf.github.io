Character substitution

(mPDF <= 4.x)

Character (font) substitution

In documents using win-1251, iso-8859-2/4/7/9 codepages, characters which are not included in the main codepage, but which are included in any of the core Adobe fonts (win-1252,  Symbols,  Dingbats) will display using the core fonts. Because they are displayed using a different font they may appear 'odd'.

This "character (font) substitution" allows a small file size, whilst making most characters at least readable. If appearance is more important than file size/speed, then consider using a UTF-8 output file.

Note: Character substitution was turned ON by default, prior to v4.0

 

A similar substitution can be enabled when using utf-8 (multibyte) documents, by setting the configuration variable in config.php:

$this->useSubstitutionsMB = true;

Any characters which do not have a corresponding glyph in the required font file, will be substituted by the appropriate character from the core Adobe fonts (win-1252,  Symbols,  Dingbats. If the characters are not included in these core fonts, they may be substituted from the default font for the document if present.

It is not recommended to enable this for regular use, as it will add to the processing time.
