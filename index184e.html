Adding new fonts (v4.x) - Archived

IMPORTANT NOTE: There was a bug in the makefonts.php file distributed with version v4.1.1 - v4.4

Only Open Source or GPL fonts have been included with mPDF for copyright reasons. You can add your own fonts to use in mPDF, but you have to create appropriate files. These instructions are for a Windows PC. If you need versions for Linux, please see TTF2PT1 and the modified version for unicode fonts by Steven Wittens at UFPDF.

On your (Windows) PC

  1. Add the .TTF font files to the [path_to_mpdf]/makefonts/ directory
  2. Rename font names if required so:
    • contain only characters a-z (lowercase only), 0-9 and _ (underscore)
    • style variants should be fontname.ttf  fontnameb.ttf  fontnamei.ttf  fontnamebi.ttf 
  3. Point your browser to  [path_to_mpdf]/makefonts/makefonts.php
  4. Follow the on-screen instructions

IMPORTANT NOTE: In steps 4 and 8 there is an option to move all the files to the appropriate font directory. This utility does NOT overwrite existing files in the font directories, and does not give any error message to indicate this (this will be corrected in mPDF 4.6).

Note: Remember to edit the file config_fonts.php to make the fonts available to mPDF (you simply need to un-comment the appropriate lines).
